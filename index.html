What is JavaScript?

  • An object-oriented scripting language with types and operators, standard built-in objects, and methods.
  • Tool for making dynamic and interactive webpages.
  • Syntax is based on the Java and C languages.

  • One main difference: functions are objects, giving functions the capacity to hold executable code and be passed around like any other object.

Popularity

Strings

Sequences of characters

                                "hello".length; // 5
"hello world".length; // 11
"Hello World!".length; // 12

More accurately, strings are sequences of Unicode characters, with each character represented by a unique numeric value that applies across platforms and programs.

Strings, cont'd

There's our first brush with JavaScript Objects!

Strings have methods just like objects that allow you to manipulate the string and access information about the string:

                                "hello".charAt(0); // "h"
"hello".charAt(4); // "o"
"hello".length; // 5
"hello, world".replace("hello", "goodbye"); // "goodbye, world"
"hello".toUpperCase(); // "HELLO"

Other Types

null - indicates a deliberate non-value

undefined - indicates an uninitialized value

var person = null; 
var x; // x has undefined type which is a constant

booleans - true/false values

Booleans

False

false, 0, empty strings (""), NaN, null, and undefined all become false

True

All other values resolve to true

You can perform this conversion explicitly using the Boolean() function: 
                                Boolean("");  // false
Boolean(234); // true

Boolean operations such as ∓& (logical and), || (logical or), and ! (logical not) are supported;

Variables

New variables in JavaScript are declared using the var keyword:

                                var a;
var name = "simon";

Numeric Operators

+ - * /

% is the remainder operator, called modulo

Values are assigned using =

Compound assignment statements += and -=

                            
a = 9 % 2;  // a = 1 
b = 6 % 3;  // b = 0

x += 5;
x = x + 5;

Operators, cont'd

++ and -- increment and decrement numeric values respectively
Can be used as prefix or postfix operators.

Concatenation

The + operator also does string concatenation: 
                        "hello" + " world"; // "hello world"

If you add a string to a number (or other value) everything is converted in to a string first.

                            "3" + 4 + 5;  // "345"
3 + 4 + "5"; // "75"

Adding an empty string to something is a useful way of converting it to a string itself.

Comparisons

< > <= >=

Work for both strings and numbers

Returns a boolean

Equality

The double-equals operator performs type coercion if you give it different types, with sometimes interesting results:

                        123 == "123"; // true
1 == true; // true
To avoid type coercion and make sure your comparisons are always accurate, you should always use the triple-equals operator: 
                            123 === "123"; // false
1 === true;    // false

There are also != and !== operators.

    Switch

    The switch statement can be used for multiple branches based on a number or string:

    switch(action) {
  case 'draw':
    drawIt();
    break;
  case 'eat':
    eatIt();
    break;
  default:
    doNothing();
}

    Switch, cont'd

    If you don't add a break statement, execution will "fall through" to the next level. This is very rarely what you want — in fact it's worth specifically labeling deliberate fallthrough with a comment if you really meant it to aid debugging:

    switch(a) {
  case 1: // fallthrough
    code here;
  case 2:
    eatIt();
    break;
  default:
    doNothing();
}

    Switch, cont'd

    The default clause is optional.
    You can have expressions in both the switch part and the cases if you like.
    Comparisons take place between the two using the === operator:

    switch(1 + 3) {
  case 2 + 2:
    yay();
    break;
  default:
    neverhappens();
}

    Objects

    Simple collections of name-value pairs.

    Similar to:

    • Dictionaries in Python
    • Hashes in Perl and Ruby
    • Hash tables in C and C++
    • HashMaps in Java
    • Associative arrays in PHP

    Everything in JavaScript is an Object

    There are two basic ways to create an empty object:

    var obj = new Object();  //

    And:

    var obj = {};  //object literal syntax; preferred: more convenient, JSON format

    Object Literal Syntax

    Can be used to initialize an object in its entirety:

    var obj = {
  name: "Carrot",
  "for": "Max",
  details: {
    color: "orange",
    size: 12 
  }
}
//name is a string, value is any JavaScript type value

    Attribute access can be chained together:

    obj.details.color; // orange
obj["details"]["size"]; // 12

    Object Prototypes

    Object prototypes can be used to set up instance creation

    function Person(name, age) {
this.name = name;
this.age = age;
}

// Define an object instance of a Person
var you = new Person("You", 24);

    Two ways to access object properties:

    obj.name = "Simon";
var name = obj.name;
// And..
obj["name"] = "Simon";
var name = obj["name"];

    Second method: good for setting and getting properties with names that are reserved words..

    obj.for = "Simon"; // Syntax error, because 'for' is a reserved word
obj["for"] = "Simon"; // works fine

    Arrays

    Arrays in JavaScript are actually a special type of object.

    Indexes vs. Length

    var a = new Array();
a[0] = "dog";
a[1] = "cat";
a[2] = "hen";
a.length; // 3

    Indexes typically start at 0

    Length is 1 greater than the index

    Arrays, cont'd

    A more convenient notation is to use an array literal:

    var a = ["dog", "cat", "hen"];
a.length; // 3

    Note that array.length isn't necessarily the number of items in the array. Consider the following:

    var a = ["dog", "cat", "hen"];
a[100] = "fox";
a.length; // 101

    Arrays, cont'd

    If you query a non-existent array index, you'll get a value of undefined returned:

    var a = ["dog", "cat", "hen"];
a[100] = "fox";
a.length; // 101

typeof a[90]; // undefined

    Arrays, cont'd

    We can use the length to iterate over an array using the following:

    for (var i = 0; i < a.length; i++) {
// Do something with a[i]
}

    This is slightly inefficient as you are looking up the length property once every loop. An improvement is this:

    for (var i = 0, len = a.length; i < len; i++) {
  // Do something with a[i]
}

    ForEach Loop

    Another way of iterating over an array that was added with ECMAScript 5 is forEach():

    ["dog", "cat", "hen"].forEach(function(currentValue, index, array) {
// Do something with currentValue or array[index]
});

    Functions

    Along with objects, functions are the core component in understanding JavaScript

    function add(x, y) {
  var total = x + y;
  return total;
}

    Takes 0 or more named parameters

    Can declare its own variables local to that function

    Can return a value, terminating the function

    Functions without return statements return undefined

    The named parameters turn out to be more like guidelines than anything else. You can call a function without passing the parameters it expects, in which case they will be set to undefined.

    add(); // NaN

    You can't perform addition on undefined

    You can also pass in more arguments than the function is expecting:

    add(2, 3, 4); // 5 
added the first two; 4 was ignored

    Array Arguments

    Functions have the 'arguments' variable, an array-like object holding all of the values passed to the function

    function add() {
  var sum = 0;
  for (var i = 0, j = arguments.length; i < j; i++) {
    sum += arguments[i];
  }
  return sum;
}

add(2, 3, 4, 5); // 14

    That's really not any more useful than writing 2 + 3 + 4 + 5 though.

    Arrays, cont'd

    Let's create an averaging function:

    function avg() {
  var sum = 0;
  for (var i = 0, j = arguments.length; i < j; i++) {
    sum += arguments[i];
  }
  return sum / arguments.length;
}

avg(2, 3, 4, 5); // 3.5

    But what if you want to find the average of an array?

    Solution

    You could just rewrite the function as follows:

    function avgArray(arr) {
  var sum = 0;
  for (var i = 0, j = arr.length; i < j; i++) {
    sum += arr[i];
  }
  return sum / arr.length;
}

avgArray([2, 3, 4, 5]); // 3.5

    Anonymous Functions

    var avg = function() {
  var sum = 0;
  for (var i = 0, j = arguments.length; i < j; i++) {
    sum += arguments[i];
  }
  return sum / arguments.length;
};

    Semantically equivalent to the function avg() form

    Anonymous Functions, cont'd

    Enable all sorts of clever tricks. Here's a way of "hiding" some local variables — like block scope in C:

    var a = 1;
var b = 2;

(function() {
  var b = 3;
  a += b;
})();

a; // 4
b; // 2

    Functions: Recap

    JavaScript functions are themselves objects — like everything else in JavaScript — and you can add or change properties on them just like we've seen earlier in the Objects section.
